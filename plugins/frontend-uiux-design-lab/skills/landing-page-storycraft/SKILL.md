# Landing Page Storycraft

Turn a feature list into a persuasive page. Each section should earn its place and move the user toward a single conversion goal.

## Story Arc

Use this default sequence unless the product clearly needs a different flow:

1. hook with a clear promise
2. immediate proof or trust
3. explain the problem and product angle
4. deepen with feature or workflow sections
5. handle objections
6. end with a decisive CTA

## Workflow

1. Decide the page's main conversion goal before touching layout.
2. Build a section list where each block has one job.
3. Make the hero immediate: promise, audience, primary CTA, and one proof cue.
4. Alternate density and breathing room so the page does not flatten into identical sections.
5. Keep copy scannable and mobile-first.

## Useful Resources

- Use `../../scripts/surface_brief.py --surface landing --brief "<prompt>"` to turn a vague request into a concrete section plan.
- Use `../../assets/ui-review-template.md` when you want a structured critique before rewriting the page.

## Strong Defaults

- Give the hero one dominant visual idea instead of stacking many competing treatments.
- Put proof early when the product is new, technical, or unfamiliar.
- Let typography and spacing create rhythm before adding more decoration.
- If the lower page sections feel repetitive, reduce the section count and increase contrast between the remaining ones.

## Guardrails

- Avoid a hero followed by four nearly identical feature card rows.
- Avoid burying the CTA below narrative sections that do not increase conviction.
- Avoid mobile layouts that turn side-by-side concepts into overly tall, repetitive stacks.
