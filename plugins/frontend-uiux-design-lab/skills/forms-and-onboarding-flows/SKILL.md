# Forms And Onboarding Flows

Design flows that reduce hesitation and make progress obvious. Every field, step, and message should help the user continue with confidence.

## Workflow

1. Identify what the user wants to achieve and what the system needs from them.
2. Remove or defer non-essential fields.
3. Group fields into logical chunks and sequence them from easy to harder.
4. Make validation timely, local, and calm.
5. Show clear progress, reassurance, and escape hatches.

## Good Defaults

- Keep labels explicit and helper text short.
- Use step-based flows only when chunking truly reduces cognitive load.
- Prefill, infer, or postpone information whenever possible.
- Explain why sensitive data is needed before asking for it.

## Guardrails

- Avoid placeholder-only labels.
- Avoid dumping every rule into a single error summary after submit.
- Avoid multi-step flows that hide how much work remains.
