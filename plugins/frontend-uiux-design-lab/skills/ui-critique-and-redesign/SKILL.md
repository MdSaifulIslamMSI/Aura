# UI Critique And Redesign

Give direct, constructive feedback and convert it into a practical redesign direction. Start with the biggest visual or interaction problems, then define the smallest set of changes that meaningfully improves the experience.

## Workflow

1. Identify what the screen is trying to help the user do.
2. State what already works so the redesign preserves useful signals.
3. Call out the biggest hierarchy, spacing, copy, state, or trust problems.
4. Translate findings into a concrete redesign direction with clear priorities.
5. Prefer a coherent design move over a long list of cosmetic tweaks.

## Output Shape

- what works
- top problems
- redesign direction
- must-change components or tokens
- risks to watch during implementation

## Useful Resources

- Use `../../assets/ui-review-template.md` to structure the critique.

## Guardrails

- Avoid vague praise that hides the main issues.
- Avoid redesign plans that replace everything without reason.
- Avoid focusing on isolated pixels while ignoring the primary task flow.
