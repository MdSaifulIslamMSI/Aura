# UX Copy IA Tuning

Tighten words and structure so users know where they are, what they can do, and what happens next. Use this skill when the product feels functionally correct but mentally noisy.

## Workflow

1. Identify the primary action and the user's likely question on each screen.
2. Rewrite headings as outcome-oriented statements instead of vague category labels.
3. Shorten labels and CTAs to familiar words with a clear verb.
4. Ensure empty, loading, success, and error states explain the next step.
5. Align navigation, filters, and section names with the user's mental model instead of internal terminology.

## Rules Of Thumb

- One promise per heading.
- Start primary CTAs with a verb when possible.
- Favor plain language over jargon or cleverness.
- Explain irreversible or high-stakes actions before the user commits.
- Keep brand voice, but do not trade clarity for personality.

## What Good Output Looks Like

- a cleaner section hierarchy
- clearer CTA priority
- shorter labels
- supportive state copy
- reduced ambiguity in navigation and filters

## Guardrails

- Avoid filling the interface with marketing copy that competes with task completion.
- Avoid stacked synonym labels such as "discover explore browse."
- Avoid empty states that only describe absence instead of offering a next move.
