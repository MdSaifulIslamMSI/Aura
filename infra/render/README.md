# Aura Render Backend + Vercel Frontend

This repo is now set up for:

- `app/` on Vercel
- `server/` on Render
- `Redis` on Render Key Value

Use the root Render blueprint:

- [render.yaml](/c:/Users/mdsai/Downloads/Kimi_Agent_Flipkart-Style%20Frontend/render.yaml)

## Target shape

- `Vercel` serves the React frontend
- `Render Web Service` runs the Express backend
- `Render Key Value` provides Redis for split-runtime workers
- `MongoDB Atlas` stays external

## Required manual envs in Render

Set these in the Render dashboard for the `aura-api` service:

- `MONGO_URI`
- `BYTEZ_API_KEY`
- `CORS_ORIGIN`
- `APP_PUBLIC_URL`

These are already handled by the blueprint:

- `REDIS_URL` from the Render Key Value instance
- `UPLOAD_SIGNING_SECRET` generated by Render
- `CRON_SECRET` generated by Render

## Free-plan behavior

On Render free:

- the web service spins down after inactivity
- the local filesystem is ephemeral

That means review-media uploads stored under `REVIEW_UPLOAD_DIR` can disappear after a restart or redeploy. This repo is therefore `Render free demo-ready`, not truly persistent on the free plan.

If you later upgrade the web service, keep using local uploads and move `REVIEW_UPLOAD_DIR` to the persistent disk mount path.

## Frontend on Vercel

Set this Vercel environment variable:

- `VITE_API_URL=https://<your-render-service>.onrender.com/api`

Then redeploy the frontend.

## Deployment options

1. `Recommended`: connect the repo to Render and let `render.yaml` create the backend service and Redis.
2. `Optional`: use the GitHub workflow:
   - [deploy-backend-render.yml](/c:/Users/mdsai/Downloads/Kimi_Agent_Flipkart-Style%20Frontend/.github/workflows/deploy-backend-render.yml)

Required GitHub secret for the workflow:

- `RENDER_BACKEND_DEPLOY_HOOK_URL`

Create that deploy hook from the Render backend service settings.
