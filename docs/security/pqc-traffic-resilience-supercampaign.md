# PQC And Traffic Resilience Supercampaign

This campaign combines two scopes:

- PQC real-target evidence and scorecard maturity.
- Traffic-resilient security fortress controls against DDoS, bot floods, abuse, expensive-route pressure, upload storms, payment abuse, AI endpoint abuse, and database exhaustion.

## Enforced Now

- PQC proof, scorecard, provider register, and real-target proof scripts.
- Traffic budgets, timeout budgets, body-size guards, load shedding, attack mode, abuse scoring, query/cache guards, and traffic proof scripts.

## Safe Defaults

- PQC real-target mode defaults to disabled.
- Traffic simulation defaults to dry-run.
- WAF examples default to staging detection mode.
- Production-impacting actions require explicit approval.

## Limits

No 100% quantum-proof claim. No 100% DDoS immunity claim. Full end-to-end PQC remains provider/browser/WebPKI capped. Massive traffic survival requires CDN/WAF/provider absorption and origin lockdown.
