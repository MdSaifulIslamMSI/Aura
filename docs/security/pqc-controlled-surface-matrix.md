# PQC Controlled-Surface Matrix

This matrix is the source of truth for Aura's practical post-quantum posture. It does not claim complete end-to-end PQC. Browser, WebPKI, hosted provider, payment, email, app-store, and third-party SDK cryptography remain ecosystem or provider controlled.

Status values:

- `enforced`: repo gate or hardened config evidence exists now.
- `lab-proven`: lab/staging proof exists, not production forced.
- `documented`: runbook and checker evidence exist.
- `provider-dependent`: Aura tracks and monitors but cannot directly change the provider crypto.
- `not applicable`: PQC is not the relevant control.

## Operating Rules

- Hybrid migration comes before any direct replacement of stable production crypto.
- OpenSSL 3.5+ native standardized PQ algorithms are preferred for lab proof where available.
- OQS Provider and liboqs remain lab/staging only unless deliberately approved with rollback evidence.
- CI fails repo-owned evidence/config gaps, not missing PQC tooling on a developer laptop.
- No production rollout should weaken existing TLS, secret scanning, route enforcement, or auth gates.
