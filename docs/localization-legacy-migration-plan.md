## Migration Order

1. Checkout/payment/auth/cart critical labels and status text.
2. Order/refund/support actions and recovery-state copy.
3. Seller onboarding and seller dashboard command labels.
4. Navigation, product detail labels, wishlist, filters, delivery, and support status copy.
5. Admin dashboard and low-risk marketing/settings/help copy.

## Safe Migration Pattern

1. Add or reuse a semantic descriptor in `app/src/i18n/messages/criticalMessages.js`.
2. Add reviewed entries to every required catalog in `app/src/i18n/messages/reviewed/`.
3. Use `intl.formatMessage(...)` for stable UI copy.
4. Keep dynamic product, seller, review, chat, and support text on the dynamic-content path only.
5. Preserve English fallback and existing market/currency/date behavior.
6. Run `npm --prefix app run i18n:check`.
7. Run focused component tests for the touched files.
8. Keep `npm run i18n:legacy-report` non-blocking until migrated folders have an allowlist.

## Runtime Translation Boundary

Runtime translation must not be used for stable checkout, auth, payment, cart,
order, or seller dashboard copy. It remains reserved for dynamic content and is
disabled/noop by default in production.
