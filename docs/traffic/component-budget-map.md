## Identity Dimensions

- Anonymous: primarily per-IP and, where useful, per-session.
- Authenticated: per-IP, per-account, and per-session.
- Admin: admin route class plus per-IP and per-user/session budgets.
- Flow/device/session: OTP reset flow tokens, WebAuthn/trusted-device challenge state, client session headers, and canonical live-call session keys.

## Monitoring Evidence

The common metric family is `aura_traffic_budget_denied_total` with registry observability tags for component, profile, and route class. Load-shedding state is exported by `aura_traffic_load_shedding_state`; abuse events are covered by the existing traffic fortress reports.
