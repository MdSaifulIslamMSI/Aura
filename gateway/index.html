Curated access layer

Choose the host. Keep the same Aura reality.

This gateway is designed as the calmer, more deliberate front door above Aura's live storefront stack. Vercel and Netlify stay separate as public runtimes, but identity, cart, wishlist, checkout, profile, and order state still resolve into the same production experience.

Desktop + mobile downloads

Install Aura on the device you actually use.

Desktop links follow the latest published desktop release. Android and iPhone links follow the latest published mobile release, while most product updates still flow into installed mobile shells through the hosted Aura storefront.

Release channels

Desktop and mobile binaries are published from GitHub Releases, then surfaced here by platform. Desktop auto-update stays pinned to the desktop release lane, while Android and iPhone use the separate mobile release lane.

Windows Windows 10/11

For everyday PC users

Choose the standard installer for most laptops and desktops. Portable builds are available when you do not want an installer.

Free Windows builds are unsigned, so Microsoft Defender SmartScreen may show a warning on first launch.

macOS Apple Silicon + Intel

For MacBooks, iMacs, and Mac minis

Use Apple Silicon for M-series Macs. Use Intel for older Macs. ZIP builds are provided for advanced workflows.

Apple Silicon DMG Intel Mac DMG Apple Silicon ZIP Intel ZIP
Linux AppImage, Deb, RPM, tar.gz

For the major Linux desktop families

AppImage is the easiest universal option. Deb is for Debian/Ubuntu families, RPM is for Fedora/RHEL families, and tar.gz is for manual installs.

x64 AppImage Deb package RPM package tar.gz archive

Android APK + AAB

For Android phones and tablets

Use the APK for direct installs on your own device. The AAB build is included for Play Console and managed distribution workflows.

Android binaries come from the dedicated mobile release lane, so they do not interfere with the desktop auto-update channel.

Android APK Android AAB
iPhone + iPad IPA + simulator

For iPhone users and Apple testers

Use the signed IPA when a real-device build is published. The simulator ZIP is available for Mac-based QA and Xcode testing workflows.

Apple device installs require Apple signing and provisioning. If the IPA is not published yet, this lane falls back to the mobile release page instead of a dead link.
